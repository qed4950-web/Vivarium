# 업무현황 운영 가이드

## 목적
- 리팩토링 사이클마다 해야 할 업무를 명확히 기록하고, 분기 및 프로젝트 마일스톤에서 완료 여부를 최신 상태로 유지한다.

## 디렉터리 구조
```
업무현황/
  cycle-01_1차리팩토링전/
    미완.md      # 진행 중/계획된 업무
    완료.md      # 완료된 업무
  cycle-02_2차리팩토링전/
    미완.md
    완료.md
  ...
```
- `cycle-XX` 번호는 리팩토링 순서에 맞춰 증가시킨다. 이름에 리팩토링 목적(예: 1차리팩토링전)도 포함해 구분한다.
- 새 사이클을 열 때 이전 사이클의 미완 항목을 검토해 필요한 것만 다음 사이클 `미완.md`에 이관한다.

## 기록 방식
- `미완.md`: 체크박스 형태로 업무를 추가한다. 예) `- [ ] 데이터 파이프라인 검증`
- `완료.md`: 완료 시 체크박스를 옮기거나 복사한다. 예) `- [x] 센서 통신 오프라인 대응 점검`
- 각 항목에 예정/완료 일자, 담당자, 참고 문서(해당 가이드 파일명) 링크를 함께 적는다.

## 업데이트 타이밍
- **매 분기 말**: 미완/완료 상태를 일괄 검토하고 최신화한다.
- **프로젝트 주요 단계(릴리스, 모델 버전업 등)**: 해당 Cycle 내부 항목을 즉시 완료 처리하여 기록을 남긴다.
- **새 리팩토링 착수 시**: 새 `cycle-XX` 디렉터리를 만들고 템플릿 파일을 복사한 뒤, 우선순위 업무를 `미완.md`에 적는다.
